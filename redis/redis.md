# 一级缓存是会话级别的缓存，主要用于减少会话内的重复查询，提升单次会话的查询性能。它的生命周期短、作用范围小，适合缓存临时数据或会话内的热点数据。

# redis执行命令还是单线程

# 多线程是基于读写，瓶颈是IO读写，不是cp

# 热key，几十万的请就去访问 某个特定的key，造成雪崩
# - 打散到不同服务器
# - 提前预热，使用二级缓存

# 缓存击穿：缓存过期导致请求直接打到db上
# - 加锁更新

# 缓存穿透：查询到了没有的数据
# - 可以将空值（如 null）缓存到缓存系统中，并设置较短的过期时间

# 缓存雪崩：某一时刻大量缓存失效
# - 不同的key设置不同的过期时间

# 过期策略 惰性删除 | 定期删除
# - 定期删除：随机取key检查
# - 惰性删除: 查询之前删除

# 惰性删除 + 定期删除 + 内存淘汰机制（兜底）
# noeviction：默认策略，当内存达到限制时，写入操作会返回错误，但读取操作仍然有效
